# 3.1

3.1 我尝试对 GCLink 进行了如下改良：

## A. 训练稳定性与收敛

### A1. 修正学习率调度

* **改动**：删除 batch 循环里的 `scheduler.step()`；在每个 epoch 末尾调用一次。
* **理由**：batch 级 StepLR 会导致学习率在一个 epoch 内衰减很多次，后期几乎不更新。
* **效果证据**：预训练 loss 明显更低、更能持续下降
  * 原版 pretrain：约 **266–270** 区间震荡
  * 优化后 pretrain：降到约 **232**（20 epoch）
> 所以我说能不能让会写代码的人来写代码！为什么会犯这种问题！

### A2. 损失函数数值稳定

* **改动**：保持 `pred` 为 logits，loss 直接用 `BCEWithLogitsLoss`。
* **理由**：更稳、更不容易出现数值/梯度问题；为 `pos_weight` 做准备。
* **效果证据**：训练过程更稳定，后续加入对比 warm-up 后 BCE 可持续下降（见 C1 的 loss 分解）。

## B. 对比学习 / 增强真正生效

### B1. 修复增强逻辑 bug

* **改动**：`GCLink.forward()` 中，encoder 输入从 `data_feature` 改为 `x1` / `x2`。
* **理由**：之前增强特征生成了但没用上，对比信号会变弱或不一致。
* **效果证据**：验证 AUC/AUPR 较原版明显提升（尤其 AUPR 早期就上升到 0.83+）。


### B2. 对比项权重策略

* **改动**：`loss = bce + lam(epoch) * con_loss`，lam 从小逐步到目标（在这个例子中我用到的是 0.1）。
* **理由**：避免对比项在前期压制监督 BCE，训练更稳。
* **效果证据**：loss 分解显示 `bce` 逐步下降、`con` 也下降；total 上升来自 lam 增加，是合理现象。

### B3. 训练日志改造

* **改动**：记录并打印平均 `total/bce/con`。
* **理由**：避免被“总 loss 上升”误导；能定位到底是哪一项导致波动。
* **效果证据**：你看到 `bce` 从约 0.622 → 0.576 逐步下降，说明监督目标确实在学。

第二，**SimSiam 在当前实验下已经表现出更好的验证性能和略优的测试性能**。

尤其是 `best_val_AUPR` 从 `0.855` 提升到 `0.861`，这个提升比前面那种 `0.001` 级别的波动更有意义；而 `test_AUPR` 也从 `0.857` 提升到了 `0.858`，虽然幅度很小，但至少说明没有以牺牲测试性能为代价换取验证集提升。

* `lam` 从 `0.001` 平滑升到 `0.020`
* `bce` 从 `0.6301` 逐步降到 `0.5974`
* `AUPR` 在中后期达到 `0.860~0.861`

这说明新的调度策略让监督主任务始终占主导，SimSiam 的一致性约束是逐步介入的，因此整体训练更平滑。相比之前较快升温的设置，这一版更符合这个任务“以边预测为主、表示学习为辅”的特点。

接下来我们尝试对他优化，结果无一例外的失败了。

在将 InfoNCE 替换为 SimSiam 后，模型整体性能已经能够达到并略优于原始基线。但在后续进一步优化过程中，并非所有改动都带来了正向收益。结合实验结果，可以将目前已经验证为**效果不佳或无明显收益**的优化路径总结如下。

### 1. 过小的图增强强度并不能带来性能提升

在当前框架下，尝试将图增强中的删边概率从原先较优设置调整为 `pe=0.10` 后，模型性能出现明显下降。实验结果显示，该设置下验证集 `best_val_AUPR` 降至 `0.849`，测试集 `AUC` 和 `AUPR` 分别降至 `0.927` 和 `0.851`，明显低于当前最优配置（`best_val_AUPR=0.861`，`test_AUC=0.929`，`test_AUPR=0.858`）。从训练过程来看，较小的删边比例虽然保留了更多原始图结构信息，但同时也削弱了双视图之间的差异性，使 SimSiam 的一致性学习目标变得不够充分，最终未能提升下游链接预测性能。因此，**减弱图增强强度不是当前任务中的有效优化方向**。

### 2. 将 SimSiam 约束范围缩小到 batch 节点会削弱整体效果

为了让辅助损失更加贴近链接预测任务，曾尝试将 SimSiam 的对比范围从“全图节点”缩小为“当前 batch 中出现的节点”。这一做法原本希望使一致性学习更聚焦于当前训练样本涉及的节点表示，但实验结果表明，该策略反而带来了轻微退化：验证集 `best_val_AUPR` 从 `0.861` 降至 `0.858`，测试集 `AUC` 和 `AUPR` 也分别下降到 `0.927` 和 `0.855`。这说明，尽管任务目标是边预测，但当前 GCLink 框架下的 SimSiam 收益仍然依赖于较充分的全图上下文信息。仅对 batch 子图节点施加一致性约束，会使表示学习过于局部化，从而削弱模型的泛化能力。因此，**“仅对 batch 节点做 SimSiam”并不是优于全图约束的方案**。

### 3. 在 edge MLP 中引入 gated 交互机制未能提升性能

针对链接预测头的表达能力不足问题，还尝试在原有 `edge_mlp` 的基础上加入 gate 机制，即使用可学习门控对节点对之间的乘积交互项进行调制，期望增强 TF-target 对之间的高阶关系建模能力。然而实验结果显示，该改动不仅没有带来性能提升，反而导致整体指标明显下降，最终仅取得 `best_val_AUPR=0.848`、`test_AUC=0.924` 和 `test_AUPR=0.849`。这表明，在当前数据规模和模型结构下，额外引入门控交互会增加 decoder 的优化难度，反而破坏了原有 edge MLP 已经较为稳定的判别能力。因此，**更复杂的边打分头并不一定优于当前较简单的 edge MLP 结构**。

### 4. 优化方向上存在“辅助模块过度复杂化”问题

综合以上实验可以发现，当前模型的性能提升主要来自于两个关键因素：一是使用 SimSiam 替代 InfoNCE 后获得了稳定的无负样本一致性学习能力；二是通过 `con_w=0.02` 与 `warmup_epochs=20` 的组合，使辅助损失能够以较平缓的方式介入训练过程。在这一基础上，无论是减弱图增强、缩小一致性约束范围，还是进一步复杂化 edge scorer，都未能继续提升模型性能，反而造成不同程度的性能退化。这说明当前模型已经在现有数据和结构条件下接近一个局部最优点，后续性能瓶颈不再主要来自 SimSiam 模块本身，而更多取决于训练稳定性与参数调度策略。


目前可以得到如下阶段性结论：

1. **SimSiam 本身是有效的替代方案**，并且在适当调参后可略优于 InfoNCE。
2. **并非所有“更强”或“更复杂”的改动都能继续提升性能**。
3. 当前已经验证失败的路径包括：

   * 将删边概率降至 `pe=0.10`
   * 将 SimSiam 损失限制在 batch 节点范围内
   * 在 `edge_mlp` 中引入 gated 交互机制
4. 因此，当前最优策略应保持为：

   * `pe=0.20`
   * `con_w=0.02`
   * `warmup_epochs=20`
   * 全图范围 SimSiam
   * 原始 `edge_mlp`
